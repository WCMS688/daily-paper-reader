## 摘要
先进的2.5D系统级封装（SiP）在高性能系统中占比日益增长。尽管封装和互连选择对整体系统设计影响重大，但系统架构师仍缺乏一个能够综合考虑这些选择的早期设计空间探索框架。当前的互连模型主要分为两类：1）详细模型，通常不够灵活且需要深厚的封装专业知识；2）高层模型，无法提供足够信息以做出准确的架构设计决策。在这项工作中，我们提出了一个自动化的芯粒IP生成框架，可为各种2.5D封装和通信配置提供功耗、性能和面积估计。该IP生成器生成高层仿真/估计、RTL仿真以及布局布线级实现所需的标准配套文件（Verilog、Liberty、LEF和数据手册）。利用我们的框架，架构师可以通过快速评估不同封装策略的功耗、性能和面积，共同优化封装和芯粒架构。作为案例研究，我们评估了跨多种封装选项生成的UCIe接口。

## Abstract
Advanced 2.5D Systems-in-Package (SiPs) compose a growing portion of high-performance systems. While the packaging and interconnect choices play a large role in the overall system design, system architects still lack a suitable framework for early design space exploration which takes these choices into account. Current interconnect models fall mostly into the categories of 1) detailed models which are generally inflexible and require deep packaging expertise, or 2) high-level models which don't provide enough information to make accurate architectural design decisions.   In this work, we present an automated chiplet IP generation framework which provides power, performance, and area estimates for various 2.5D packaging and communication configurations. The IP generator produces standard collaterals required for high-level simulation/estimation, RTL simulation, and place-and-route-level implementation (Verilog, Liberty, LEF, and datasheet). Using our framework, architects can co-optimize the package and chiplet architecture through rapid power, performance, and area estimates of various packaging strategies. As a case study, we examine generated UCIe interfaces across several packaging options.

## 2. 论文提出的方法论

### 核心思想
- 以**单一JSON配置文件**驱动全流程，自动生成物理信道RC模型、终端/均衡选择、收发器协同优化、面积模型，并输出Liberty、Verilog、LEF、SPICE、SDC等标准交付件。

### 关键技术细节

#### 2.1 物理信道模型
- 采用**分布式π-梯形RC模型**（见图2），将信道分解为：芯片焊盘、微凸点、中介层焊盘、走线（三段π-梯形）、ESD保护等部件。
- 每个部件的R和C由几何参数（凸点间距、走线宽度、层厚等）和材料常数通过解析公式计算，例如：
  - 焊盘电容：平行板电容模型 \( C_{\text{pad}} = \varepsilon_0 \varepsilon_r w_p^2 / t_{ox} \)
  - 凸点电阻：直流+趋肤效应 \( R_{\text{bump}} = \sqrt{R_{DC}^2 + R_{AC}^2} \)
- 走线RC从文献查表值缩放（硅中介层：185 fF/mm, 1.04 Ω/mm；有机基板：138 fF/mm, 0.036 Ω/mm）。
- 用Elmore延迟和能量公式进行快速预过滤，最终时序和功耗通过SPICE/Liberate获得。

#### 2.2 自动链路适配
- **终端**：基于UCIe规范边界表，根据链路距离/数据速率自动选择四级终端强度（无→轻→标准→强），采用Thevenin split拓扑避免直流偏置电流。
- **被动均衡**：基于信道在奈奎斯特频率的RC低通损耗，自动选择五级被动均衡强度（无→轻→标准→强→激进），均衡零点和极点由RC推导。

#### 2.3 收发器协同优化
- **TX架构**：CMOS反相器链，几何级数分扇出，非反相极性。
- **RX架构**：两级（小预放大器 + 大输出缓冲器），最小化输入电容。
- **N×N帕累托搜索**：
  - 分别对TX和RX进行表征：TX以输出负载为变量扫描，RX以输入转换时间为变量扫描。
  - 对每个(TX_i, RX_j)组合，通过查表插值获得延迟和能量，避免穷举SPICE仿真。
  - 在满足UCIe延迟预算（16 UI）的可行解中，提取非支配帕累托前沿，提供三种选择策略：平衡、最佳功耗、最佳延迟。

#### 2.4 面积模型与LEF生成
- 基于计算的晶体管尺寸、被动均衡电容、终端电阻、ESD二极管等，结合工艺密度常数，计算每通道面积。
- 支持用户提供凸点映射（bump map），生成带引脚位置的LEF抽象。

#### 2.5 多PDK可移植性
- 内置TSMC 16nm FinFET、TSMC 65nm、GF 45nm SOI的完整配置，封装了工艺差异（FinFET fin count、电容密度、金属层厚度等）。
- 移植新工艺只需编写JSON配置和Liberate模板。

### 算法流程（文字说明）
1. 用户输入JSON配置（封装类型、距离、凸点间距、数据速率、通道数、PDK等）。
2. 基于几何参数和物理常数，计算信道RC，生成π-梯形SPICE子电路。
3. 根据信道RC和数据速率，自动选择终端和均衡等级。
4. 执行TX/RX协同优化：尺寸搜索→Liberate表征生成查找表→查表匹配所有组合→帕累托过滤→选择最优设计。
5. 计算面积，生成凸点映射和LEF。
6. 输出Liberty、Verilog、LEF、SPICE、CSV等交付件。

## 6. 论文的主要结论与发现

1. **封装选择的关键影响**：有机基板在较长距离（~10 mm以上）反而比硅中介层具有更低的能量/比特和更低的延迟。硅中介层的低凸点电容优势被高走线电阻迅速抵消，在超过~10 mm后，有机基板的粗走线低电阻占主导。
2. **自动帕累托优化有效**：CLIPGen能自动生成帕累托前沿，使架构师可在能量和延迟之间权衡。
3. **跨节点对比**：TSMC 16nm因低供电电压（0.8V）在能量上最优，GF 45nm在延迟上最优，但能量更高。
4. **校准有效性**：在30 mm有机基板校准点上，框架结果与ISSCC 2026原型实测点吻合（图5中金色星号位于仿真曲线附近）。

## 7. 优点

- **高自动化**：从简单JSON配置到完整EDA交付件，无需封装专长。
- **多PDK支持**：内置三种常见工艺，易于扩展。
